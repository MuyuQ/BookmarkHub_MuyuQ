## 1. 开发环境搭建

### 1.1 系统要求

| 要求 | 版本 |
|------|------|
| Node.js | >= 18.x |
| npm | >= 9.x 或 pnpm >= 8.x |
| 浏览器 | Chrome 88+ / Firefox 109+ |

### 1.2 安装依赖

```bash
# 克隆仓库
git clone https://github.com/dudor/BookmarkHub.git
cd BookmarkHub

# 安装依赖
npm install
# 或使用 pnpm
pnpm install
```

### 1.3 启动开发服务器

```bash
# Chrome 开发模式
npm run dev

# Firefox 开发模式
npm run dev:firefox
```

开发服务器启动后：
1. 打开 Chrome，访问 `chrome://extensions/`
2. 开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 `.output/chrome-mv3` 目录

## 2. 代码规范

### 2.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类/接口/类型 | PascalCase | `BookmarkInfo`, `SyncData` |
| 枚举值 | PascalCase | `BrowserType.CHROME` |
| 函数/变量 | camelCase | `performSync()`, `githubToken` |
| Boolean 变量 | 前缀 is/has/enable | `isSyncing`, `enableAutoSync` |
| 常量 | UPPER_SNAKE_CASE | `STORAGE_KEYS`, `MV3_CONFIG` |
| 私有成员 | 前缀 _（可选） | `_fetchRemoteData` |

### 2.2 代码风格

```typescript
// ✅ 正确示例
import { Setting } from './setting';
import { BookmarkInfo } from './models';
import { logger } from './logger';

// 缩进：2 spaces
// 字符串：单引号

if (setting.enableAutoSync) {
  startAutoSync();
}

// ❌ 错误示例

```

## 4. 反模式与禁止操作

### 4.1 API 使用

```typescript
// ❌ 禁止：直接使用 chrome API
chrome.bookmarks.getTree()

// ✅ 正确：使用 browser API（跨浏览器兼容）
browser.bookmarks.getTree()
```

### 4.2 消息处理

```typescript
// ❌ 禁止：异步消息缺少 return true
browser.runtime.onMessage.addListener((msg, sender, sendResponse) => {
    performSync().then(result => sendResponse(result));
    // 缺少 return true，响应会丢失
});

// ✅ 正确：异步消息必须 return true
browser.runtime.onMessage.addListener((msg, sender, sendResponse) => {
    performSync().then(result => sendResponse(result));
    return true;  // 必需！
});
```

### 4.3 类型安全

```typescript
// ❌ 禁止：使用 as any 或 @ts-ignore
const data = response as any;
// @ts-ignore
data.someProperty;

// ✅ 正确：使用 unknown + 类型断言
const data: unknown = response;
if (typeof data === 'object' && data !== null && 'someProperty' in data) {
    const typedData = data as { someProperty: string };
}
```

### 4.4 设置访问

```typescript
// ❌ 禁止：直接访问 optionsStorage
const options = await optionsStorage.getAll();

// ✅ 正确：使用 Setting.build()
const setting = await Setting.build();
console.log(setting.githubToken);
```

### 4.5 安全相关

```typescript
// ❌ 禁止：日志输出敏感数据
console.log(`Token: ${setting.githubToken}`);
console.log(`Password: ${setting.webdavPassword}`);

// ✅ 正确：敏感数据不输出日志
logger.info('Settings loaded', { hasToken: !!setting.githubToken });

// ❌ 禁止：生产环境保留 console.log
console.log('Debug info');

// ✅ 正确：使用 logger 系统（生产环境自动过滤）
logger.debug('Debug info');
```

## 8. 常见问题

### 8.1 开发服务器不启动

**症状**: `npm run dev` 报错

**解决方案**:
```bash
# 清除缓存重新安装
rm -rf node_modules .output
npm install
npm run dev
```

### 8.2 扩展加载失败

**症状**: Chrome 显示「manifest.json 无效」

**解决方案**:
- 确保 WXT 已正确初始化：`npm run postinstall`
- 检查 `.output` 目录是否存在
- 验证 `wxt.config.ts` 配置正确

### 8.3 同步失败

**症状**: 书签同步不工作

**排查步骤**:
1. 检查 GitHub Token 权限（需要 gist 管理）
2. 检查 Gist ID 是否正确（32或40位十六进制）
3. 检查 Gist 是否为私有
4. 查看 Console 日志定位错误

### 8.4 WebDAV 连接失败

**症状**: WebDAV 同步失败

**排查步骤**:
1. 验证 URL 格式（需包含完整路径）
2. 检查用户名密码是否正确
3. 测试 WebDAV 服务器是否可用
4. 检查 CORS 配置（如果跨域）
